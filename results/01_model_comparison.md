# 모델 비교·선정 보고서

## 1. 비교 목적

업무 메일 초안 작성 과업에 가장 적합한 LLM을 선정한다.

## 2. 동일 입력

아래 프롬프트를 GPT-5, GPT-5 mini, Claude Haiku 4.5에 동일하게 입력한다.

```text
너는 업무 메일 초안 작성 도우미다.
아래 규칙과 예시를 참고해 메일 초안을 작성해줘.

[규칙]
- 제공된 사실만 사용한다.
- 모르는 사실은 만들지 않고 확인 필요 항목에 적는다.
- 출력 형식은 제목/본문/확인 필요 항목을 지킨다.
- 최종 답변에는 장문의 추론 과정을 쓰지 않는다.

[예시 1]
입력:
[목적] 회의 일정 변경 요청
[수신자] 외부 협력사 담당자
[상황] 내부 검토 지연으로 기존 일정을 변경해야 함
[핵심 내용] 기존 6월 3일 10시, 요청 6월 5일 14시 또는 16시
[톤] 정중하고 간결
[금지 표현] 과도한 사과
[필수 포함 항목] 변경 사유, 대체 일정, 회신 요청

출력:
제목: 회의 일정 변경 요청드립니다
본문:
안녕하세요.
내부 검토 일정 조정으로 기존 6월 3일 오전 10시 회의를 6월 5일 오후 2시 또는 4시로 변경 요청드립니다.
두 시간 중 가능하신 일정을 회신 부탁드립니다. 어려우시면 가능한 시간을 알려주시면 다시 조율하겠습니다.
확인 필요 항목:
- 상대방 가능 시간

[예시 2]
입력:
[목적] 검토 자료 요청
[수신자] 사내 디자인팀
[상황] 제안서 작성을 위해 최신 화면 이미지가 필요함
[핵심 내용] 금요일 오전까지 전달 요청, 어려우면 가능 일정 공유 요청
[톤] 협조 요청
[금지 표현] 재촉하는 표현
[필수 포함 항목] 필요한 자료, 희망 기한

출력:
제목: 제안서용 화면 이미지 요청드립니다
본문:
안녕하세요.
제안서 작성을 위해 최신 화면 이미지 자료를 요청드립니다.
가능하시다면 금요일 오전까지 공유 부탁드립니다. 일정상 어려우시면 전달 가능한 시점을 알려주시면 반영하겠습니다.
확인 필요 항목:
- 제공 가능한 자료 범위
- 전달 가능 일정

[예시 3]
입력:
[목적] 사과 메일 작성
[수신자] 고객
[상황] 문제가 있었음
[핵심 내용] 빨리 처리한다고 말하기
[톤] 알아서
[금지 표현]
[필수 포함 항목]

출력:
제목: 확인 필요
본문:
메일 초안 작성 전에 아래 정보가 필요합니다.
1. 어떤 문제가 발생했는지
2. 고객에게 안내 가능한 조치 내용과 예상 일정
3. 원하는 톤: 정중한 사과, 간단한 안내, 재발 방지 약속 중 선택
확인 필요 항목:
- 문제 내용
- 조치 범위
- 안내 가능한 일정

[실제 입력]
[목적]
외부 협력사에 일정 변경을 요청한다.

[수신자]
외부 협력사 담당자

[상황]
내부 검토가 예상보다 늦어져 6월 3일 오전 회의를 6월 5일 오후로 변경하고 싶다.

[핵심 내용]
- 일정 변경 사유는 내부 검토 지연
- 기존 일정: 6월 3일 오전 10시
- 요청 일정: 6월 5일 오후 2시 또는 오후 4시
- 상대방이 어렵다면 가능한 시간을 회신해 달라고 요청

[톤]
정중하고 간결한 비즈니스 톤

[금지 표현]
핑계처럼 들리는 표현, 과도한 사과

[필수 포함 항목]
제목, 본문, 확인 필요 항목

[출력 형식]
제목:
본문:
확인 필요 항목:
```

## 3. 비교 대상

| GPT-5 | x1 | 웹 | 2026-05-28 | 웹 기본 설정 |
| GPT-5 mini | x0.5 | 웹 | 2026-05-28 | 웹 기본 설정 |
| Claude Haiku 4.5 | x0.5 | 웹 | 2026-05-28 | 웹 기본 설정 |

## 4. 평가 축

| 평가 축 | 설명 |
|---|---|
| 사실 충실성 | 기존 일정, 요청 일정, 변경 사유를 빠짐없이 반영하고 없는 정보를 만들지 않는가 |
| 톤 적합성 | 외부 협력사에 맞는 정중하고 간결한 비즈니스 톤인가 |
| 형식 준수 | 제목, 본문, 확인 필요 항목 형식을 지키는가 |
| 간결성 | 불필요한 사과, 장황한 설명, 중복 표현 없이 바로 발송 가능한가 |
| 확인 필요 처리 | 상대방 확인이 필요한 항목을 명확히 분리하는가 |

## 5. 평가 결과

| 모델 | 사실 충실성 | 톤 적합성 | 형식 준수 | 간결성 | 확인 필요 처리 | 근거 요약 |
|---|---:|---:|---:|---:|---:|---|
| GPT-5 | 5 | 5 | 5 | 5 | 5 | 입력 사실을 모두 반영했고, 문장이 가장 짧고 바로 발송 가능함. 확인 필요 항목도 구체적으로 분리함. |
| GPT-5 mini | 5 | 5 | 5 | 4 | 5 | 사실과 형식은 정확함. 다만 제목에 기존 날짜를 넣고 본문에 감사 인사를 추가해 GPT-5보다 약간 길어짐. |
| Claude Haiku 4.5 | 4 | 5 | 5 | 4 | 4 | 전체 품질은 좋지만 요청 일정을 본문 첫 문장에서는 "6월 5일 오후"로 축약한 뒤 다음 문장에서 보완함. 확인 필요 항목도 다소 포괄적임. |

## 6. 모델별 결과 요약

- GPT-5: 세부 일정, 변경 사유, 회신 요청을 모두 반영했다. 문장이 가장 간결하고 확인 필요 항목을 구체적으로 분리했다.
- GPT-5 mini: 사실 반영과 형식 준수는 우수했다. 다만 제목과 본문이 GPT-5보다 약간 길어져 간결성에서 1점 낮게 평가했다.
- Claude Haiku 4.5: 정중한 톤과 형식은 잘 지켰다. 다만 요청 일정을 처음에 넓게 표현하고 확인 필요 항목이 포괄적이라 사실 충실성과 확인 필요 처리에서 감점했다.

## 7. 최종 선정 결론

최종 모델은 GPT-5로 선정한다.

업무 메일 초안 작성은 제공된 사실을 정확히 보존하고, 수신자에게 바로 보낼 수 있을 만큼 간결해야 한다. GPT-5는 기존 일정, 요청 일정, 변경 사유, 회신 요청을 모두 빠짐없이 반영했다.

또한 확인 필요 항목을 구체적으로 분리해 사용자가 발송 전 확인할 부분을 바로 파악할 수 있었다. GPT-5 mini도 품질은 높았지만 문장이 조금 더 길었고, Claude Haiku 4.5는 일부 정보를 포괄적으로 표현했다.

따라서 정확성, 간결성, 확인 필요 처리까지 모두 안정적인 GPT-5가 이 과업에 가장 적합하다.

## 8. 무료 모델만 사용할 때의 운영 전략

무료 모델만 사용해야 한다면 아래 방식으로 품질을 보완한다.

- 입력 템플릿을 고정해 목적, 수신자, 상황, 핵심 내용, 톤, 금지 표현을 빠짐없이 제공한다.
- 출력 형식을 제목/본문/확인 필요 항목으로 강제한다.
- 사실, 일정, 정책, 승인 여부는 확인 필요 항목으로 분리하게 한다.
- 발송 전 사용자가 날짜, 금액, 승인 여부, 첨부 여부를 직접 확인한다.
